## 目标

在现有长报告（`ReportArtifact`）基础上增加一页式执行摘要（`ExecutiveSummaryArtifact`），让律师快速定位核心信息。

## 生成器逻辑（纯规则层，零 LLM 调用）

### `top5_decisive_issues`
- 按 outcome_impact 降序排序：high=3 > medium=2 > low=1 > None=0
- 取前 5 个 issue_id

### `top3_immediate_actions`
- 若 `action_recommendation` 为 None → `"未启用"`，`source_recommendation_id=None`
- 否则收集优先级顺序的 action item IDs：
  1. `claims_to_abandon[].suggestion_id`（最紧迫）
  2. `recommended_claim_amendments[].suggestion_id`
  3. `evidence_supplement_priorities[]`（gap_ids）
  4. `trial_explanation_priorities[].priority_id`
- 取前 3 个，`source_recommendation_id = action_recommendation.recommendation_id`

### `top3_adversary_optimal_attacks`
- 从所有提供的 `optimal_attack_chains` 收集 `top_attacks[].attack_node_id`
- 取前 3 个
- `source_attack_chain_ids = [c.chain_id for c in optimal_attack_chains]`

### `current_most_stable_claim`
- 从 `amount_calculation_report.claim_calculation_table` 派生文本：
  - 找 delta=0 的稳定诉请，生成描述文本
  - 若无 delta=0 则描述 delta 最小的诉请
  - `source_amount_report_id = amount_calculation_report.report_id`

### `critical_evidence_gaps`
- 若 `evidence_gap_list` 为 None → `"未启用"`
- 否则按 `roi_rank` 升序排序，取前 3 个 `gap_id`

## Inline Plan Review

**风险与决策**：

1. **Union[list[str], Literal["未启用"]] 类型**：Pydantic v2 支持 `Union[list[str], str]`；Literal["未启用"] 需要 Annotated 或直接用 str 加 validator。设计决策：使用 `Union[list[str], str]` + validator 强制非"未启用"情况必须是 list。

2. **source 字段**：额外添加 `source_recommendation_id`, `source_attack_chain_ids`, `source_amount_report_id` 以满足"可回连"要求。

3. **top5_decisive_issues 最大数量**：严格取 min(5, len(issue_list))，不填充空字符串。

4. **current_most_stable_claim 生成策略**：delta=0 的最稳定诉请描述 → 有 delta != 0 时描述偏差 → 无 claim_calculation_table 时描述一致性状态。始终返回非空文本。

5. **模块放置**：`engines/report_generation/executive_summary_generator/`（report_generation 阶段产物）。
