# v1.5 程序化庭前会议规格（已实现）

> 状态：✅ COMPLETED（合并到 main，2026-03-31）
> 实现文档见：`engines/pretrial_conference/`

本文档为 v1.5 规格占位，详细实现参见架构文档 `02_architecture.md`。

## 核心能力

- `evidence_state_machine`：证据状态迁移（`submitted → challenged → admitted`）
- 质证维度：真实性、关联性、合法性、证明力
- 法官发问机制：只追问未闭合争点，只能引用已进入程序的材料
- 输出：《模拟庭前会议纪要》《质证焦点清单》《法官可能追问 Top10》

## 验收口径（已达成）

- 法官追问中，70% 以上能被律师评价为"确实会问"
- 未经允许的 `private` 证据泄漏为零
- 所有裁判倾向都能回溯到某轮质证记录
- 证据状态只允许合法迁移
